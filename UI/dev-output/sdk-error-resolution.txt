ROOT CAUSE ASSESSMENT
--------------------------------------------------------------------------------

1. cookiesToSet implicit any
   - SDK/package: @supabase/ssr createServerClient callback does not provide explicit types for the setAll(cookiesToSet) parameter in its type definitions.
   - App config: TypeScript strict mode (noImplicitAny) in tsconfig causes the error.
   - Verdict: COMBINATION — Supabase SSR types are loose; app uses strict TypeScript. Not a bug in sdk-web template per se; template assumes standard Next.js + Supabase setup. Fix: add explicit types at call site (app-specific).

2. useSearchParams Suspense
   - Next.js App Router requires useSearchParams() to be wrapped in <Suspense> when used in a page that can be statically prerendered.
   - sdk-web template: The console page in the template may not have included this pattern if created before Next.js enforced it, or if the template was built when useSearchParams was used differently.
   - Verdict: SDK TEMPLATE GAP — Template should be updated to wrap any page using useSearchParams in Suspense by default. This is a Next.js requirement, not optional.
