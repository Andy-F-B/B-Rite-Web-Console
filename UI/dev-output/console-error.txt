# Console Page — Issues & Fix

## Issues Encountered

### 1. Client-side exception (initial)
- **Symptom:** Clicking Editor button caused "Application error: a client-side exception has occurred"
- **Cause:** createClient() calls getSupabaseConfig() which throws when NEXT_PUBLIC_SUPABASE_URL or NEXT_PUBLIC_SUPABASE_ANON_KEY are missing. On Vercel without env vars, the page crashed with a generic error.
- **Mitigation:** Added try/catch in app/console/page.tsx to show a friendly Configuration Error message instead of crashing.

### 2. Vercel build error
- **Symptom:** Build failed on Vercel
- **Cause:** Missing env vars. lib/env.ts throws if vars are missing or still placeholder.
- **Fix:** Add NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY to Vercel → Settings → Environment Variables for all environments (Production, Preview, Development).

### 3. Configuration Error persists after env setup
- **Symptom:** User added env vars to Vercel, redeployed, but console page still showed "Missing NEXT_PUBLIC_SUPABASE_URL"
- **Initial diagnosis:** NEXT_PUBLIC_* vars are inlined at build time; redeploy without clearing cache might reuse old artifacts.
- **Added:** /api/debug-config endpoint — returns { hasUrl, hasKey } to verify server has vars. If server has vars but client fails, suggested clearing build cache and redeploying.

### 4. Root cause — dynamic env access (actual fix)
- **Symptom:** Even after redeploy without cache, console still failed. /api/debug-config showed hasUrl/hasKey true (server had vars) but client bundle did not.
- **Root cause:** lib/env.ts used `process.env[key]` — dynamic key access. Next.js only inlines static references like `process.env.NEXT_PUBLIC_SUPABASE_URL`. Dynamic lookups (process.env[variable]) are NOT inlined at build time and remain undefined on the client.
- **Fix:** Changed lib/env.ts to use static property access:
  - Before: `const value = process.env[key]` (dynamic — never inlined)
  - After: `const url = process.env.NEXT_PUBLIC_SUPABASE_URL` and `const anonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY` (static — inlined at build time)

## Summary

The console page failed in production because getSupabaseConfig() used dynamic process.env[key] access, which Next.js cannot inline. The bundler replaces only literal `process.env.VARIABLE_NAME` references. Switching to static access allows the values to be baked into the client bundle at build time.
