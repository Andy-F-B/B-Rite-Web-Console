================================================================================
SDK-WEB — SETUP PROCEDURE
================================================================================
Folder structure, wiring, and b-rite integration per AI App Development Handbook V2.

================================================================================
1. MASTER FOLDER LAYOUT (HANDBOOK 2.1)
================================================================================

When a project is initialized, the following structure is created. Paths in [ ] 
are the TARGET locations — where files live at runtime. The SDK holds TEMPLATES 
that get installed to these locations.

repo/
  app/                    # Next.js pages
  components/             # UI components
  lib/                    # Utilities, Supabase clients, query functions
    env.ts
    supabase.ts
    supabase-server.ts
    database.types.ts
    schemas/              # Zod validation schemas
  hooks/
  styles/
  middleware.ts

  db/
    schema.sql            # Current full schema (always up to date)
    migrations/          # 001_initial_schema.sql, 002_add_characters.sql, ...

  ai/                     # Master AI instruction and spec files
    brain.md
    b-rite/               # Core language engine
    help/
    sdks/
    plugins/
    tasks/
    context/
    specs/                 # SDK spec outputs (product.md, architecture.md, etc.)
      product.md
      architecture.md
      database.md
      ui.md
      api.md
      rules.md
      security.md
      debug.md
      checklist.md
      lessons.md
      migrations.md
      migration_checklist.md

  ai/context/             # Short anchor summaries
    product_anchor.md
    architecture_anchor.md
    database_anchor.md
    security_anchor.md

  ai/tasks/               # One file per feature
    example_feature.md

  UI/                     # HTML previews (mobile_preview.html, responsive_preview.html)
  UI/dev-output/          # Plans, docs, txt returns
  feature-packs/          # Reusable modules (auth-system, notifications, teams, ...)

================================================================================
3. WIRING — WHERE OUTPUTS GO
================================================================================

| Content Type           | SDK Source              | Target at Runtime      |
|------------------------|-------------------------|------------------------|
| Spec templates         | sdk-web/specs/          | [root/ai/specs/]       |
| Context anchors        | sdk-web/context/        | [root/ai/context/]      |
| Task templates         | sdk-web/tasks/         | [root/ai/tasks/]       |
| HTML previews          | (generated by AI)       | [root/UI/]             |
| Plans, docs, txt       | (generated by AI)       | [root/UI/dev-output/]  |
| Generated app code     | (generated by AI)       | [root/app/], [root/components/], etc. |

The SDK does NOT create [root/UI/] or [root/UI/dev-output/] — those are project 
folders. Variables.br defines paths so AI knows where to write outputs.

================================================================================
4. B-RITE INTEGRATION
================================================================================

a) REGISTER SDK
   Add to [root/ai/b-rite/sdk-registry.br]:
     003 = [root/ai/sdks/sdk-web/]

b) INVOKE SDK
   *dev* !sdk|003| br : { run; <sdkWebSetup> } :
   *dev* !sdk|003| br : { run; <specPhase>; 'my app idea' } :

c) BRAIN BLOCK (optional)
   Copy sdk-web-brain-block.md content into brain.md for persistent context.
   Or rely on !sdk|003| to load it per-script.

d) VARIABLES
   Read [root/ai/sdks/sdk-web/variables.br] for paths.
   Key paths: aiSpecs → [root/ai/specs/], uiPreviews → [root/UI/], output → [root/UI/dev-output/]

e) FUNCTIONS
   Read [root/ai/sdks/sdk-web/functions.br] for function index.
   All phase functions callable via run; <functionName>.

================================================================================
5. INSTALL FLOW (install.br)
================================================================================

1. Copy sdk-web folder to [root/ai/sdks/sdk-web/] (if not present)
2. Add 003 = [root/ai/sdks/sdk-web/] to sdk-registry.br
3. (Optional) Paste sdk-web-brain-block.md into brain.md
4. Update brain.md installed sdks: sdk-web — [root/ai/sdks/sdk-web/]
5. Run *dev* !sdk|003| br : { run; <sdkWebSetup> } : to verify

================================================================================
6. NEW PROJECT SETUP
================================================================================

When creating a NEW web project from scratch:

1. Run <specPhase> with app idea → generates [root/ai/specs/] spec files
2. Run <anchorPhase> → generates [root/ai/context/] anchors
3. Run <taskPhase> for each feature → generates [root/ai/tasks/] task files
4. Run <migrationPhase> → generates [root/db/migrations/]
5. Run <buildPhase> → generates app/, components/, lib/
6. Run <auditPhase> → security and checklist verification
7. Run <previewPhase> → generates [root/UI/] HTML previews

Templates in sdk-web/specs/, context/, tasks/ are used as defaults when the 
target folders are empty. Once a project has its own specs, the SDK reads 
those instead.

Templates in sdk-web/specs/ install to [root/ai/specs/]
